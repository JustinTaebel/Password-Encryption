How passwords work

Passwords are used for authentication and access control in digital systems. When creating a password, users choose a unique combination of characters. To secure passwords, they are transformed using a one-way hashing function, often with the addition of a unique salt. The resulting hash is stored, not the actual password. During authentication, the provided password is hashed and compared to the stored hash. If they match, access is granted. Password security can be improved through complexity requirements, length, regular changes, and multi-factor authentication. It's important to use strong, unique passwords, avoid reusing them, and be cautious of phishing attempts to enhance security.

How encryption works

Encryption is a process that converts readable data into unreadable form to protect sensitive information. It involves applying an encryption algorithm and a key to the plaintext, resulting in ciphertext. The encryption key is necessary to reverse the process and retrieve the original data, known as decryption.

How to Use

Password Checker

Enter in a password that is at least 6 characters long, contains one capital letter, and one symbol. Strength of password is determined based off of a point system that gives points for length, number of Capitals, numbers, and symbols. Once you put in a strong password, you have the option to encrypt.

Encryption

The submitted password is encrypted by going through a simple ceasar cipher algorithm with a randomly set shift variable and then displays the encrypted string. You can then decrypt it and it will replace the encrypted string with the original plain text string. If you encrypt the same password again, a different encrypted string will display, but the same password will be decrypted.
